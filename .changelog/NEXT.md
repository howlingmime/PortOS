# Next Release

## Added

- **Global Command Palette (`⌘K`) with shared backbone for the voice agent** — the `CmdK` overlay has been upgraded from a content-search box into a true command palette. Typing fuzzy-matches across every navigable page (Dashboard, Brain, CoS sub-pages, MeatSpace tabs, Settings, etc.) and a curated set of voice-agent actions (Capture to Brain, PM2 status, Today's health summary, Current time, Feed digest, Recent Brain entries, etc.). Enter navigates or runs the action; arrow keys walk the list; Esc closes. The existing Brain/Memory/Apps/History/Health content search is preserved underneath as a second section so nothing regresses. Empty-query state shows a curated shortlist (Dashboard, Brain Inbox, Tasks, Goals, Review Hub, Capture to Brain, time, goal list, today's summary) so `⌘K` + Enter always does something useful.

  - **DRY backbone** — navigation is now defined once in `server/lib/navManifest.js` (`NAV_COMMANDS` + `resolveNavCommand` fuzzy resolver) and consumed by both the voice agent's `ui_navigate` tool and the palette via `GET /api/palette/manifest`. The old hand-synced 140-line `NAV_PAGES` map inside `server/services/voice/tools.js` is gone — adding a page to the manifest now updates both the sidebar's spoken/typed entrypoint AND the palette automatically, eliminating the "keep in sync with Layout.jsx" drift footgun.
  - **Palette-safe action whitelist** — `POST /api/palette/action/:id` dispatches a narrow subset of voice tools (brain/goals/meatspace/pm2/feeds/time/daily-log); DOM-driving tools (`ui_click`, `ui_fill`, `ui_select`, `ui_check`, `ui_navigate`) are intentionally excluded since the palette has no voice-agent DOM context to drive. Action descriptions + JSON Schema parameters are lifted from the same voice-tool registry, so palette hints and voice hints can never drift apart.
  - **Tests** — new `server/lib/navManifest.test.js` (9 tests covering shape invariants + fuzzy-match tiers + alias map) and `server/routes/palette.test.js` (5 tests covering manifest shape, action dispatch, whitelist enforcement). All 30 existing voice-tool tests continue to pass against the refactored resolver.
  - **Keyboard help modal updated** — the `?` shortcut list now advertises "Open command palette" instead of "Open search" and describes Enter as "Go to page, run action, or open result".

  Files: `server/lib/navManifest.js` (new), `server/routes/palette.js` (new), `server/services/voice/tools.js` (refactored), `server/index.js` (mounts `/api/palette`), `client/src/services/apiPalette.js` (new), `client/src/components/CmdKSearch.jsx` (palette behavior), `client/src/components/KeyboardHelp.jsx` (updated copy). Tests: `server/lib/navManifest.test.js`, `server/routes/palette.test.js`.

- **Customizable Dashboard with Saved Layouts** — the Dashboard is no longer a static hardcoded widget list. A widget registry (`client/src/components/dashboard/widgetRegistry.jsx`) describes each tile (`{ id, label, Component, width, gate? }`) and the Dashboard renders the active layout's ordered widget list over a 12-column grid. Four built-in layouts ship: **Everything** (the original), **Focus** (quick-task + cos + upcoming), **Morning Review** (alerts + upcoming + review-hub + goals + death-clock), **Ops** (system-health + cos + backup + apps + quick-stats). New layouts can be created via the layout picker → "Edit layouts…" → Save as new. Widgets can be added/removed per layout, reordered with ↑/↓ buttons (keyboard- and touch-accessible), and gated so data-dependent widgets (streak, hourly activity, quick stats) auto-hide when there's nothing to show. The Apps tile is intentionally un-gated — it renders its own "Add App" empty-state CTA on a blank install. Persisted to `data/dashboard-layouts.json` via `GET/PUT/DELETE /api/dashboard/layouts`; built-in layouts are protected from deletion. Inline-state delete/duplicate confirmations — no `window.confirm`/`prompt`.

  **Palette integration**: `⌘K` synthesizes one command per layout ("Dashboard: Focus", "Dashboard: Ops", etc.) so users can switch layouts from anywhere with a keystroke. Layouts re-fetch on every palette open so newly-created or renamed layouts show up without a page remount.

  Files: `server/services/dashboardLayouts.js` (new), `server/routes/dashboardLayouts.js` (new), `server/routes/dashboardLayouts.test.js` (new, 7 tests), `server/index.js` (mounts route), `client/src/services/apiDashboard.js` (new), `client/src/components/dashboard/{widgetRegistry.jsx,LayoutPicker.jsx,LayoutEditor.jsx,builtins/*}` (new), `client/src/pages/Dashboard.jsx` (fully rewritten), `client/src/components/CmdKSearch.jsx` (layouts section), `CLAUDE.md` ("Dashboard Widgets & Layouts" section with widget-entry contract).

## Changed

- **Command palette cleanup pass** — post-review simplify: `normalizeLabel` is exported once from `navManifest.js` and imported by `voice/tools.js` (was duplicated verbatim); `getToolMetadata(id)` added to `voice/tools.js` so `routes/palette.js` no longer reaches through the OpenAI-shaped function spec; palette manifest frozen at module load (`buildActionManifest` no longer runs per request) with `Cache-Control: private, max-age=300`. Client scorer precomputes lowercased `_label`/`_aliases`/`_keywords`/`_section` so per-keystroke fuzzy matching does zero re-lowercasing over the 117-entry manifest. `CmdKSearch.jsx` trimmed ~80 lines: collapsed the triple `useMemo` chain, merged `renderCommandRow` + inline search-result row into a single `renderRow`/`renderGroup` pair, flattened `dispatchCommand` into a `DISPATCH` lookup table, dropped the dead `running` state, removed two `.catch()` error-swallows (errors now propagate to the central toast handler per CLAUDE.md), cleared `resultRefs` on close. `ALIAS_KEYS` cached once in `resolveNavCommand`. Decorative section-banner comments and narration comments removed across the new files.
